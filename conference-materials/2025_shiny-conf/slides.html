Death By Dropdown?

A Developer’s Guide To Building Dashboards That Won’t Fry Your Client’s Brain

Milena Eickhoff & Jeremy Winget, PhD

The Problem

Ever felt like this?

  • Endless dropdown menus
  • Overwhelming complexity
  • Users disengaged
  • Delayed decision-making
  • “Dashboard rot”

It Doesn’t Have To Be This Way


By shifting the focus from cramming data to crafting stories


::: {.r-fit-text} We can guide users effortlessly to insights

:::

The BID Framework

A 5-Stage Behavior Insight Design Framework

  1. Notice the Problem
  2. Interpret the User’s Need
  3. Structure the Dashboard
  4. Anticipate User Behavior
  5. Validate & Empower the User
  • This framework distills psychological principles into actionable steps
  • Each stage builds on the previous one
  • The framework is grounded in behavioral science, communication theory, and UX design

Where BID Fits In

  • Builds on established UX heuristics and theories
  • Incorporates data storytelling frameworks
  • Systematically integrates cognitive bias mitigation
  • Cross-disciplinary synthesis (UX, data viz, behavioral economics)
  • Stage-based process rather than heuristic lists
  • Specifically tailored for analytics dashboards
  • BID doesn’t reinvent the wheel but packages existing knowledge into a structured process
  • Makes bias anticipation an explicit step for designers
  • Brings together multiple disciplines to create a comprehensive framework

Stage 1: Notice the Problem

Detect high cognitive friction points

  • Cognitive Load Theory (Sweller, 1988)
  • Hick’s Law (Hick, 1952)
  • Visual Hierarchies (Tufte, 1983)

Actionable Example:

# Instead of:
selectInput("var1", "Select Variable:", choices = names(data))
selectInput("var2", "Select Variable:", choices = names(data))
selectInput("var3", "Select Variable:", choices = names(data))

# Consider:
radioButtons("primaryVar", "Primary Metric:", 
             choices = c("Revenue", "Conversion", "Engagement"))

Stage 2: Interpret the User’s Need

Understand the story users need

  • Data Storytelling Framework (Matei & Hunter, 2021)
  • Processing Fluency (Alter & Oppenheimer, 2009)
  • Emotion & Fluency Effects (Song & Schwarz, 2009)

Ask key questions:

  • What insight are they trying to gain?
  • What decisions need to be made?
  • What is the simplest way to present this information?

Actionable Example:

# Instead of showing all data at once, guide the user:
fluidRow(
  box(title = "Key Business Question", 
      "Are we on track to meet Q2 targets?",
      status = "primary"),
  box(valueBoxOutput("mainKPI"))
)

Stage 3: Structure the Dashboard

Build with intentional structure

  • Principle of Proximity (Gestalt)
  • Dual-Processing Theory (System 1 & 2)
  • Default Effect (Johnson & Goldstein, 2003)
  • Aesthetic-Usability Effect (Norman, 2002)

In Practice:

  • Group related elements visually
  • Provide quick summary cues AND detailed views
  • Use smart defaults to reduce friction
  • Ensure visual consistency and appeal

Stage 3: Structure In Action

library(reactable)

# Applying Gestalt principles and defaults
reactable(
  data,
  defaultPageSize = 5,
  defaultSorted = list(revenue = "desc"),
  groupBy = "department",
  columns = list(
    revenue = colDef(
      format = colFormat(prefix = "$", separators = TRUE),
      style = function(value) {
        # System 1 color cues
        if (value > 1000000) {
          color <- "#008000"  # Green for good
        } else if (value < 500000) {
          color <- "#B22222"  # Red for concerning
        } else {
          color <- "#000000"  # Default
        }
        list(color = color, fontWeight = "bold")
      }
    )
  )
)

Stage 4: Anticipate User Behavior

Consider biases & heuristics

  • Anchoring Effect (Tversky & Kahneman, 1974)
  • Framing & Loss Aversion (Tversky & Kahneman, 1981)
  • Confirmation Bias (Nickerson, 1998)
  • Belief Perseverance (Ross & Lepper, 1980)

Mitigation strategies:

  • Provide context for all numbers
  • Frame data positively when appropriate
  • Show alternative views/analyses
  • Include “disconfirming” visualizations

Stage 4: Anticipate In Action

# Dealing with anchoring and framing
tabsetPanel(
  tabPanel("Performance", 
    fluidRow(
      box(
        title = "Sales Performance",
        plotOutput("salesPlot")
      ),
      box(
        title = "Context",
        # Provide context to prevent anchoring
        "Industry average: $1.2M",
        "Company target: $1.5M",
        "Historical range: $0.9M - $1.8M"
      )
    ),
    # Alternative view to combat confirmation bias
    checkboxInput("showAlternative", "Show alternative analysis")
  ),
  tabPanel("What If?", 
    # Encourage exploration of contrary hypotheses
    sliderInput("scenario", "If market drops by:", min = 0, max = 30, value = 10),
    plotOutput("scenarioPlot")
  )
)

Stage 5: Validate & Empower

Close the loop with clarity

  • Peak-End Rule (Kahneman et al., 1993)
  • Beautiful-Is-Good Stereotype (Dion et al., 1972)
  • Cooperation & Coordination (Tindale & Kluwe, 2015)

Implementation:

  • Strong, clear summary sections
  • Visually appealing final views
  • Collaborative features for teams
  • Exportable insights

Stage 5: Validate In Action

# Peak-End Rule and Beautiful-Is-Good implementation
fluidRow(
  box(
    title = "Key Insights",
    status = "primary", 
    solidHeader = TRUE,
    width = 12,
    # Provide clear summary
    HTML("<h3>3 Key Takeaways:</h3>
         <ol>
           <li>Revenue is up 12% against target</li>
           <li>Western region showing 3x growth</li>
           <li>Product X is underperforming by 8%</li>
         </ol>"),
    # Enable collaboration
    actionButton("share", "Share Report", icon = icon("share")),
    downloadButton("downloadReport", "Export PDF")
  )
)

After BID:

  • Actionable insights

The outcome:

Users make faster, better decisions with less cognitive effort

Practical Examples

Death By Dropdown ➡️ Decision By Design

  • Grouped Headers: Organize complex data into logical hierarchies
  • Dynamic Drilldowns: Start with big picture, then reveal details
  • Inverse Color Scales: Draw attention to what matters most
  • Narrative Pathways: Guide users through a data story

Implementation Demo

library(reactable)
library(echarts4r)

# Before: Overwhelming dropdown experience
# selectInput("region", "Region:", choices = regions)

# selectInput("timeframe", "Time:", choices = timeframes)
# selectInput("segment", "Segment:", choices = segments)

# After: Dynamic exploration with narrative focus
reactable(
  data,
  groupBy = "category",
  defaultExpanded = TRUE,
  columns = list(
    value = colDef(
      name = "Performance",
      cell = function(value, index) {
        # Contextual visualization
        sparklinePlot <- sparkline(data$trend[index], 
                                   lineColor = ifelse(value > target, "#28a745", "#dc3545"))
        div(sparklinePlot, style = "width: 100px; height: 30px;")
      }
    )
  ),
  details = function(index) {
    # Drill-down with context
    htmltools::div(
      echarts4r::e_charts(detail_data[detail_data$id == data$id[index], ]) |>
        e_line(x = "date", y = "value") |>
        e_tooltip() |>
        e_title(paste("Detailed View:", data$name[index]))
    )
  }
)

Key Takeaways

  1. Notice where users struggle with cognitive load
  2. Interpret the story they need, not just the data they have
  3. Structure dashboards to guide attention and reduce friction
  4. Anticipate biases and heuristics that affect decisions
  5. Validate with clear summaries and next steps
